Автопрайсер и P&L для маркетплейса

Система, которая по фактическим деньгам маркетплейса находит, какие из тысяч товаров зарабатывают, а какие тихо жгут бюджет — и переоценивает каталог так, чтобы каждый активный товар был в плюсе.

В демо — 8 товаров. В рабочей версии тот же расчёт идёт по ~5500 позициям: себестоимость и комиссии подтягиваются из Ozon API, цены обновляются по расписанию.

01 · задача

Магазин на ~5500 товаров, один оператор. На вопрос «этот товар продаётся в плюс или в минус?» не было ответа ни по одной позиции: маркетплейс удерживает около половины цены, сам навешивает скидки, а часть расходов списывает на уровне аккаунта, не привязывая к конкретным товарам. Управлять этим вручную невозможно.

02 · как решал

Собрал систему на Python, которая тянет из API маркетплейса фактические финансовые транзакции — начисления, комиссии, логистику, возвраты, штрафы — и сводит их в P&L по каждому проданному товару. Дальше позиции классифицируются: прибыльные, тонкие, убыточные из-за цены, убыточные из-за возвратов. Для переоценки считается «пол цены» от закупки с реальной, а не заявленной комиссией.

Любое изменение сначала проходит dry-run: система показывает план «было → станет», и только после подтверждения человеком пишет цены обратно, сохранив снимок для отката.

03 · что оказалось сложным

Напрашивалось закрытие. Вместо этого я сверил расчёт с фактическими транзакциями — и картина оказалась противоположной. Разбор вскрыл систематические перекосы: реальная комиссия заметно ниже заявленной, реальная логистика — существенно ниже максимальных тарифов, а аналитический эндпоинт недосчитывал продажи против финансового.

Отсюда принцип, зашитый в систему: прогнозная модель — только для будущих цен, финансовый результат — только из фактических транзакций.

Вся оценка конкурентности была построена на неверном из них. После сверки отправлений с начислениями правило стало жёстким: экономика каждого товара обязана сходиться в плюс на собственном начислении, без учёта внешних скидок маркетплейса.

04 · результат
  • ~5500 товаров в каталоге, 8,5 месяцев истории сведены в P&L по каждой позиции.
  • Найдены и отключены скрытые утечки на уровне аккаунта, съедавшие весь товарный плюс.
  • Обнаружены десятки товаров с патологической возвратностью (до 87% возвратов на позицию) — сняты или нейтрализованы заградительной ценой.
  • За один день переоценено 1561 товар; крупнейшая запись — 1304 цены за раз, ноль ошибок, результат верифицирован обратным чтением.
  • Инцидентов «скрипт положил магазин» не было: dry-run → откат → ручной гейт → верификация.
